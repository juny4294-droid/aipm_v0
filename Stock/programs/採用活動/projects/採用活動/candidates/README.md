# 候補者別フォルダ

候補者ごとに **1フォルダ** を切り、その中にオファー案、面接メモ、キャリアシート要約、添付などをまとめます。

## 置き場所

```
projects/採用活動/candidates/<YYYY-MM-DD_氏名>/
```

## フォルダ名（必須）

**作成日（フォルダを切った日）＋名前** とする。

| ルール | 例 |
|--------|-----|
| `YYYY-MM-DD_姓名`（姓名はスペースなし推奨） | `2026-04-12_川口泰生` |

作成日は **その候補者用フォルダを初めて作った日** を入れる（後から別日に追加した場合は、その日付で新規フォルダを切る運用でも可。チームで統一してください）。

**注意**: 個人情報を含むため、Gitに載せない運用の場合は `.gitignore` で `candidates/*/` を除外し、テンプレのみ追跡する方法もあります。

## 新規候補者を追加するとき

1. `_template` をコピーし、フォルダ名を **`YYYY-MM-DD_氏名`** にする
2. フォルダ内の `README.md` の見出し・**フォルダ作成日**・候補者サマリを記入
3. オファー案・原稿は `offer/`、その他資料は `attachments/` などに配置

## オファー文章作成プロジェクトとの関係

[オファー文章作成](../オファー文章作成/README.md) で作るレター原稿も、原則 **該当候補者フォルダ** 内（例: `offer/`）に保存します。

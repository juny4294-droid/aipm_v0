# 技術専門（IC）職位ガイドライン

## 1. 職位の位置づけ

技術専門（IC）track の職位は、次の2つの軸で貢献する。

- **レベル軸での貢献**  
  シニアエンジニア → スタッフエンジニア → プリンシパルエンジニア → ディスティングイッシュド／フェロー の段階で、技術の深さ・戦略的影響の範囲を広げる。組織横断の技術課題の解決、標準・方針の策定、技術方向性の主導を担う。
- **領域軸での貢献**  
  特定ドメイン（セキュリティ、プラットフォーム／SRE、データ／ML、品質／テスト等）の専門家・オーナーとして、その領域で組織・事業を支える。レベル軸と組み合わせて運用する（例：プリンシパルセキュリティエンジニア）。

## 2. 役割と責務

- 特定領域で自律して設計・実装し、周囲に技術的影響を与える。
- 複数チーム／プロダクトにまたがる技術課題を解決し、標準や方針を決める。
- 組織全体の技術方向性・アーキテクチャを主導する。
- 中長期の技術戦略に参画し、技術リスクとトレードオフの判断を担う。
- 業界・社外も含めた技術リーダーとして、戦略・イノベーションを牽引する（レベルに応じて）。
- 領域特化の場合は、当該領域（セキュリティ、プラットフォーム、データ、品質等）の方針・設計・改善を主導する。

## 3. 期待される能力（ガイドライン）

- **技術**: 設計・実装の質、技術トレードオフの判断、新技術・プロセスの習得と適用。
- **分析**: 課題の構造化、根因分析、優先順位づけ、技術リスクの評価。
- **対人**: 技術的合意形成、ドキュメント・発信による影響、メンタリング（レベルに応じて）。

## 4. 運用上の留意点（例）

- レベル軸と領域軸の組み合わせ（例：スタッフセキュリティエンジニア）を、評価・キャリア制度で明示する。
- IC track とマネジメント track の役割分担（意思決定権限、評価対象）を組織内で共有する。
- プリンシパル以上は、組織横断の技術方針・標準への参画機会を確保する。

## 🔶 はじめに

「経営から、病院を変える。病院から、医療を変える。」

当社は、400床以上の大病院を中心に、「医療機関向けSaaSや伴走型コンサルティング」を提供するスタートアップです。

医療の質と持続可能性が問われ、医療機関の「変革」を支援するパートナーが求められている今、メダップでは上記ビジョンを掲げ、医療×Vertical SaaSの領域で、病院の経営改善ひいては医療の持続可能性という社会課題に向き合っています。

当社が掲げるビジョンや事業の意義の詳細については、是非[メダップの目指す世界（note）](https://note.com/medup/n/nd9adbea78242)をご覧ください。

## 🔶 メダップのこと、事業のこと

### 医療×SaaSで、“病院経営”に真正面から向き合う。

データ統合基盤を武器に、病院経営のDX・地域医療の持続可能化を支援しています。

当社は、情報技術を活用してサービスを多く立ち上げ、無理のない質の高いオペレーションを日本中の医療機関・医療従事者に広めることを目指しています。これまで、病院経営の改善と地域医療の向上により貢献できる事業運営のため、積極的に資金調達を行ってきました。直近では、2024年3月にプレシリーズBで3.8億円を、2025年3月にはベンチャーデットにて1.3億円を調達し、累計約14億円の資金を確保して事業を拡大しています。

### 病院特化型AIワークフローで、業務改革をリードする

複雑だけどルール化された業務が多い病院で、**AIワークフローを活用した業務改革**——それが、いまメダップが重点的に進める事業方針です。

### コンパウンドSaaS × 伴走型コンサル

foro CRM 等のSaaSとCS/PSによる伴走を組み合わせ、**「プロダクト × データ × コンサル」**で病院経営の課題にアプローチしてきた基盤です。

## 🔶 募集背景

事業フェーズが進むなかで、**開発の土台（品質・セキュリティ・運用・標準化）**への要求水準が上がっています。

「高速での新規プロダクト開発」と「既存SaaSの安定運用・改善」——**両方**の開発が走るなか、個人の頑張りに依存せず、チームとして継続的に回る仕組みを整えるために部門横断の**テックリード**を募集します。

### ❑ 新規開発と既存プロダクト開発、両方を支える土台が必要

顧客課題の深掘りからプロダクト化までを一気通貫で進めるチームと、既存SaaSの安定運用と改善を担うチーム。どちらも**AI駆動開発**を前提に少人数で高い実行速度を目指す一方、共通のレビューの型・品質ゲート・セキュリティの回し方が揃っていないと、スピードと信頼性の両立が難しくなります。

### ❑ 既存プロダクトの技術品質とセキュリティを、チームで回せる形に

foro CRM を中心とした既存プロダクトは、病院の業務に直結する基幹に近い領域です。設計・技術負債・リリース判定・セキュリティ運用を、**担当者個人の経験**に頼り切らず、チームが再現できる形に引き上げたいと考えています。

### ❑ AI駆動開発の「型」を、部門全体に定着させる

顧客課題への新規取り組みで立ち上げた進め方を、既存プロダクトの定常開発にも浸透させる。プロンプト・テンプレ・レビュー効率化などを含む**共通基盤**を設計し、「作って終わり」ではなく**運用として定着**させることが、本ポジションの中心です。

🚩 アーリーフェーズならではの裁量で、技術の土台をゼロから設計し、事業成長に直結させたい方を募集しています。

## 🔶 ポジション概要

> （Notion: gray callout）**部門共通のAI駆動開発の土台を整え、プロダクトの技術品質とセキュリティを担うテックリードです。**
>
> 共通基盤を「作って終わり」にせず、運用として定着させます。速度と品質・セキュリティのトレードオフを言語化しながら前に進めます。

### ❑ ミッション

- 新規・既存どちらの開発チームも使える、AI駆動開発の共通ルールと仕組みを設計し、日常の開発に定着させる
- 新規開発で試行錯誤した進め方を、既存プロダクトの定常開発でも再現できる形に広げる
- 設計・技術負債・運用の判断軸をそろえ、技術品質をチームで維持・改善できる状態にする
- セキュリティの基準と対応フローを整備し、開発・運用の現場に組み込む

## 🔶 業務内容

開発の「作る」だけでなく、**チームが継続的に高品質で回る共通基盤・ルール・運用**を、部門横断でリードします。

### 🏗 AI駆動開発環境（部門共通）

❑ **概要**：少人数×AI駆動で、新規開発と既存プロダクト開発の両チームが開発できる共通の土台をつくる。

- 開発の生産性/品質を上げる共通基盤（CI、レビューの型、テンプレ、運用ルール）の設計・導入・定着
- チームへの浸透（運用設計、支援、改善ループ）
- 新規開発と既存プロダクト開発それぞれの進め方との接続（役割分担の明確化）

### 🔧 既存プロダクトの技術品質

❑ **概要**：設計・開発に技術的なフォローを行い、判断軸をチームで再現できる形にする。

- 設計レビュー/技術選定の支援、判断軸の整備
- 技術負債の見える化と返済計画（優先度づけ、ロードマップへの組み込み）
- 可観測性（ログ/メトリクス）や運用改善の推進

### 🛡 セキュリティ

❑ **概要**：プロダクト開発・運用フェーズで、セキュリティ担保をリードする。

- 基準の定義（何を守るか）と、開発プロセスへの組み込み（設計レビュー、リリース判定等）
- 脆弱性/インシデントの運用設計（受付、優先度、対応、再発防止）
- 開発チーム・インフラ担当等との分担・連携の設計

### 🤝 連携・型化

❑ **概要**：個人の頑張りではなく、チームの運用に落とし込む。

- 速度×品質/セキュリティのトレードオフを言語化し、合意形成する
- 横断的な改善のたたき台を作り、継続的に回す

## 🔶 このポジションの魅力

### 開発の土台（品質/セキュリティ/運用）を横断で設計・定着できる

プロダクト開発の「作る」だけでなく、AI駆動開発の共通ルール・仕組みと、品質・セキュリティ・運用の土台を、**部門横断**で設計し、チームの日常に定着させられるポジションです。

### 医療×Vertical SaaSで、社会的意義の大きい領域に関われる

病院経営の変革を支えるプロダクトの技術品質とセキュリティを担い、医療の持続可能性という社会課題に直接貢献できます。

### アーリーフェーズならではの裁量で、仕組みをゼロから作れる

既存の型がない状態から、AI駆動開発の進め方・レビューの型・セキュリティ運用などを設計し、組織に根付かせる経験が得られます。

### こんな体験ができます

- AI駆動開発の共通基盤を整備し、チーム全体の生産性と品質を引き上げる
- セキュリティ基準と運用フローを立ち上げ、開発プロセスに組み込む
- レビューの型や品質ゲートを設計し、チームの開発標準として定着させる

## 🔶 主な使用技術

foro CRM 等のプロダクト開発・共通基盤で主に扱う技術スタックです。

- **フロントエンド**: TypeScript, React, Next.js, Zustand, React Hook Form, Zod, OpenAPI Generator, Sass, vanilla-extract, storybook, …etc
- **バックエンド**: TypeScript, Node.js, Fastify, tRPC, Prisma, Kysely, fp-ts, typebox, Python ※1, (Rails ※2), …etc
- **データベース**: PostgreSQL, BigQuery
- **その他**: GCP (Cloud Run, AlloyDB, Cloud SQL, Cloud Build, …etc), Azure (AADB2C), Docker, Sentry, SendGrid, Figma, Cursor 等のAIコーディングツール, …etc

※1 Webアプリケーションでは TypeScript / Node.js がメイン。データ基盤構築で Python を採用しています。  
※2 以前の外注開発で使用していた Rails が一部で動いていますが、内製化後は TypeScript / Node.js へ随時切り替えています。

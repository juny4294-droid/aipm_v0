# FDE（採用）転職サイト・返信テンプレ

> **用途**: [FDE(採用)_ファーストコンタクト.md](./FDE(採用)_ファーストコンタクト.md) 送付後、候補者から返信があったときの **2通目以降の返し**（ローカル専用）  
> **送信者**: 開発部門責任者の山田  
> **JD**: [FDE(採用)_要件.md](./FDE(採用)_要件.md) / [Notion JD](https://www.notion.so/3731da5a7f3b812988a3d137b74dc772) / [採用ハブ](https://medup.notion.site/recruit)

---

## 1. この文書の位置づけ

| 文書 | 役割 |
|------|------|
| `FDE(採用)_ファーストコンタクト.md` | 初回スカウト（1通目） |
| **本ファイル** | 返信後の返し（2通目〜） |
| `candidates/.../offer/` | 選考後の入社オファー（対象外） |

**ゴール**: カジュアル面談の日程確定 → 面談後のフォロー。質問には短く答えつつ、会話に進める。

**文字数**: 2通目は初回より短くてよい（目安 **500字以内**）。媒体に上限があればカウンタで確認。

---

## 1-1. 装飾・文体

初回と同じ（[ファーストコンタクト §1-1](./FDE(採用)_ファーストコンタクト.md#1-1-媒体別の装飾コピペ時の注意)）。

- プレーンテキスト。太字・色・HTMLなし
- URLは1行単位
- 件名欄は使わない（返信スレッドにそのまま貼る）
- 初回の長い会社説明は **繰り返さない**（相手は読んでいる）

**この段階では書かない**: 確定年収・入社日・内定・オファー条件

## 2. 返信パターンの選び方

相手の返信を読んで、下表から **1つ** 選ぶ。迷ったら **A（前向き）**。

| パターン | 相手の様子 | 使うテンプレ |
|--------|------------|--------------|
| **A** | 興味あり・話したい（明確にOK） | §3-A（短尺・標準） |
| **A'** | 前向きだが説明も添えたい | §3-A詳 |
| **B** | 具体的な質問あり（業務・給与・リモート・医療未経験など） | §3-B ＋ §4の該当Q |
| **C** | 日程候補を提示してきた | §3-C |
| **D** | 検討中・タイミング尚早・忙しい | §3-D |
| **E** | お断り・見送り | §3-E |
| **F** | 業務委託希望だが社員採用のみ／即転職不要でもマッチ感あり | §3-F |
| **G** | 技術要件に不安・CS/PSの仕事に関心（FDE＝CS/PS＋エンジニアリング） | §3-G継続 |
| **H** | カジュアル面談後のお礼・前向きに検討する | §3-H |

## 2-1. 参考：実際に使った例（吉田さん・別ポジション）

FDE以外の採用で送った文。トーン・構成の参考。**FDEでは §3-F**（送信者・URLは山田用に差し替え）。

```text
ご返信ありがとうございます！

正直にお伝えすると、今回のポジションは社員としての採用を想定しており、業務委託でのご依頼は現時点では難しい状況です。せっかくお返事をいただいたのに恐縮です。

一方で、これまでのご経歴を拝見し、本当にマッチしていると感じております。すぐの転職でなくて全く問題ありませんので、中長期の選択肢の一つとして、ぜひ一度お話しできればと思っています。

まずはお互いを知る場として、カジュアルにお話しさせていただけませんか？無理にお誘いするものではなく、情報交換のような感覚で大丈夫です。

もしよろしければ、下記よりご都合のよい日程をお選びください。
https://medup.eeasy.jp/naoki.yoshida/saiyou

お返事お待ちしています！
```

**この例の型**: ①感謝 → ②正直に条件のズレ（委託不可）＋恐縮 → ③マッチ感は維持 → ④即転職不要・中長期OK → ⑤低圧カジュアル → ⑥日程URL

## 3. 返信テンプレ（コピー用）

括弧は埋める。`（日程調整URL）` は山田用 eeasy 等に差し替え（未設定なら [カジュアル面談フォーム](https://docs.google.com/forms/d/e/1FAIpQLSd_4bYRD_Kh_6CXY4F0zgdxeoPwroQA0lvxM6AD44DbRLGyTA/viewform)）。

### 3-A. 前向き・日程調整（短尺・標準）

相手が「ぜひ」「話してみたい」など明確にOKのとき。**§2-2ベース**。初回で十分説明済みならこちらを使う。

```text
（氏名）さん、ご返信ありがとうございます！
ぜひお話させてください。

下記よりご都合のよい日程を選択いただくか、（日程の条件：例「19日以降」）で候補日をお知らせいただければ幸いです。
（日程調整URL）
```

`（日程の条件）` が不要なら → `候補日をいくつかお知らせいただければ調整します。` に差し替え。

**謙遜あり（「お役に立てるか不明」等）** — §3-Aの2行目のあとに1文足す:

```text
選考というより、事業やFDEの役割、（氏名）さんのご経験について相互に情報交換する場にできれば十分です。
```

### 3-A詳. 前向き・カジュアル打診（説明も添える）

初回が短かった・相手の返信が曖昧なとき。面談のイメージを1段落足す。

```text
（氏名）さん、ご返信ありがとうございます！開発部門責任者の山田です。

お忙しいところ読んでいただき、嬉しいです。
ぜひ一度、30〜45分ほどオンラインでカジュアルにお話しできればと思います。FDEの役割や、いまメダップで進めていること、（氏名）さんのご経験との接点を率直にお話しできれば十分です。選考というより、相互にフィット感を確かめる場にしたいです。

よろしければ、下記からご都合のよい日程をお選びください。
（日程調整URL）

ご都合の合わない場合は、候補日時をいくつかいただければ調整します。
どうぞよろしくお願いいたします。
```

### 3-B. 質問への短答 ＋ カジュアル打診

`（質問への回答）` に §4 から該当ブロックを貼る。複数質問は箇条書き風に改行で並べてよい。

```text
（氏名）さん、ご返信とご質問ありがとうございます。山田です。

（質問への回答）

詳しくはカジュアルでお話しするのが早いと思います。30〜45分、オンラインでいかがでしょうか。
（日程調整URL）

ほかにも気になる点があれば、そのままメッセージでも大丈夫です。よろしくお願いいたします。
```

### 3-C. 日程候補が来たとき

```text
（氏名）さん、ご返信ありがとうございます。山田です。

ご提示いただいた（候補日時の要約：例「6/10 19:00〜」）で問題ありません。こちらでカレンダー招待をお送りします（または、下記URLから確定をお願いします）。
（日程調整URL）

当日は（氏名）さんのご経験や、FDEとしてどう関わりたいかを中心に、ざっくばらんにお話しできればと思います。よろしくお願いいたします。
```

### 3-D. 検討中・タイミング尚早

```text
（氏名）さん、ご返信ありがとうございます。山田です。

ご状況、承知しました。無理に急いでいただく必要はまったくありません。
もしタイミングが合いそうになったときは、いつでもお声がけください。募集概要は引き続きこちらです。
https://www.notion.so/3731da5a7f3b812988a3d137b74dc772

またご連絡いただける日を楽しみにしています。
```

### 3-E. お断り

```text
（氏名）さん、ご返信ありがとうございます。山田です。

ご検討いただきありがとうございました。今回は見送りとのこと、承知しました。
またご縁がありましたら、ぜひお声がけください。良いご縁がありますように。
```

### 3-F. 業務委託希望・社員採用のみ（§2-1ベース・FDE用）

相手が副業・業務委託・フリーランス希望で返信してきたとき。**委託は断りつつ、関係は残す**。即転職を前提にしない。

```text
（氏名）さん、ご返信ありがとうございます！開発部門責任者の山田です。

正直にお伝えすると、今回のFDEポジションは社員としての採用を想定しており、業務委託でのご依頼は現時点では難しい状況です。せっかくお返事をいただいたのに恐縮です。

一方で、これまでのご経歴を拝見し、FDEとして本当にマッチしていると感じております。すぐの転職でなくて全く問題ありませんので、中長期の選択肢の一つとして、ぜひ一度お話しできればと思っています。

まずはお互いを知る場として、カジュアルにお話しさせていただけませんか？無理にお誘いするものではなく、情報交換のような感覚で大丈夫です。

もしよろしければ、下記よりご都合のよい日程をお選びください。
（日程調整URL）

お返事お待ちしています！
```

**3-F と 3-D の違い**: Dは「今は忙しい・検討中」全般。Fは **雇用形態のミスマッチ** を先に正直に伝えるとき。

### 3-G. 技術要件に不安・CS/PSの仕事に関心（小林さん型）

FDEの技術Mustに届かないかも、と書いてきたうえ、**CS/PS領域を主に伺いたい**と言ってきたとき。

**前提**: 今回のスカウトは **FDE採用**。FDEはCS/PS領域の仕事も含むが、**CS/PSのみの別枠採用** とは別のご連絡。候補者への文面では「CS/PS採用候補外」と拒絶的に書かない。

**FDEの見方（山田）**: 現時点の技術スキルが要件に届いていなくても、**足りない領域をキャッチアップしながら挑戦する意志**があれば十分あり得る。初のFDEは型を一緒に作る側面も大きい（面接観点の「キャッチアップの早さ」も [FDE(採用)_要件.md](./FDE(採用)_要件.md) 下書き参照）。

**分岐**:
- CS/PS採用担当で **要件外** だが、FDEは継続検討したい → **§3-G継続**（小林さん・標準）
- FDE・CS/PSいずれも見送り → **§3-G区切り**

---

#### 3-G継続. CS/PS関心あり・技術懸念あってもカジュアル可（小林さん・確定版）

CS/PS領域への関心は **FDEの仕事と重なる** ことを先に伝える。技術ギャップは「今すぐ満たしているか」より「キャッチアップして挑戦するか」で判断。

```text
（氏名）さん

ご返信ありがとうございます。開発部門責任者の山田です。
率直にお書きいただき、ありがとうございます。

CS/PS領域について主に伺いたいとのこと、よく分かります。補足ですが、今回お声がけしているFDEは、CS/PSだけのポジションではなく、CS/PSとエンジニアリングの両方を担う職種です。現場での課題深掘りや運用定着、価値実現までの伴走といった、（氏名）さんが関心をお持ちの領域もFDEの仕事に含まれます。CS/PSチームと一体で進めながら、プロダクト化・実装・効果確認まで技術面でもリードする——その両方がFDEの役割です。

そのうえで、技術要件との距離感について率直に感じておられる点も、よく分かります。私としては、現時点ですべての要件を満たしているかどうかより、足りない領域があってもキャッチアップしながら挑戦する意志があるかどうかを、むしろ重視しています。初のFDEという性質上、一緒に進め方を形にしていく側面も大きいです。

もしよろしければ、CS/PS領域のご関心も含め、FDEの役割や（氏名）さんのご経験との接点を、オンラインで30〜45分、情報交換の感覚で率直にお話しできればと思います。技術面の懸念も含め、遠慮なくお話しできれば十分です。

（勤務地の一文：例「大阪在住で頻繁な出社が難しい点については、日常はオンライン中心で、フルリモートも応相談可能です。ただし顧客先への訪問や打合せはあります。」）

ご興味があれば、下記より日程をお選びいただくか、候補日をお知らせください。
（日程調整URL）

よろしくお願いいたします。
```

---

#### 3-G区切り. FDE・CS/PSいずれも見送り

CS/PS要件外に加え、FDEも継続する理由がないとき（関心がCS/PSのみ・キャッチアップ意欲も見えない等）。**カジュアルは打たない**。

```text
（氏名）さん

ご返信ありがとうございます。開発部門責任者の山田です。
率直にお書きいただき、ありがとうございます。

CS/PS領域については、社内の採用担当とも確認したうえで、現時点では当社のCS/PS採用要件との接点は限定的である、と判断しております。今回のご連絡はFDE採用が目的のため、CS/PSとしてのご縁をお繋ぎするのは難しい状況です。

メダップの取り組みへのご関心は、本当にありがたく思っています。またのご縁がありましたら、ぜひお声がけください。良いご縁がありますように。
```

**書かないこと**: 具体的な不足理由・他社比較・CS/PS担当の生コメントの転載。

---

### 3-H. カジュアル面談後のお礼返し（相澤さん型）

面談後に「お時間ありがとうございました」「前向きに検討します」と返ってきたとき。**押しつけず**、質問の窓口を残す。次の選考ステップへの催促はしない（社内で動くときは別途）。

**実例**: 相澤さん（2026-06）—— 面談後お礼・前向きに検討。

```text
（氏名）さん

こちらこそ、お時間をいただきありがとうございました。開発部門責任者の山田です。
（氏名）さんとお話しできたこと、大変嬉しく思っています。

前向きにご検討いただけるとのこと、ありがとうございます。ご自身のペースで大丈夫ですので、気になる点やご不明なことが出てきましたら、いつでもメッセージください。

引き続き、どうぞよろしくお願いいたします。
```

**山田側で次に進めたいとき**（相澤さんの返信には書かない。社内判断後に別メッセージで）: 正式選考の案内・追加面談の打診など。

## 4. よくある質問への短答（§3-B用）

JDの範囲で答え、細部はカジュアルへ。**コピペ後、相手の質問に合わせて不要な行は削る。**

### 業務内容・FDEの役割

```text
FDEは、CS/PSとともに病院の現場課題を深掘りし、プロダクト化・実装・効果確認まで一気通貫で進める役割です。当社にとって初のポジションなので、顧客デリバリー現場を主戦場にしながら「メダップのFDE」の進め方を一緒に形にしていくイメージです。
```

### 給与・待遇

```text
給与レンジや評価の考え方は募集概要に記載しています（経験・スキルに応じてレンジ内で協議）。カジュアルでは確定オファーではなく、レンジ感と役割のイメージをお伝えします。
https://www.notion.so/3731da5a7f3b812988a3d137b74dc772
```

### リモート・勤務地

```text
基本は東京オフィス勤務を想定していますが、顧客先への訪問・伴走も業務に含みます。リモート可否はチーム体制と案件次第なので、カジュアルで具体的にお話しできればと思います。
```

### 医療・病院ドメイン未経験

```text
医療ドメイン未経験でも問題ありません。大切なのは、顧客課題を深掘りしてプロダクトに落とし込み、実装・効果まで届けた経験です。CS/PSがドメイン知見も持っているので、一緒に現場に入り込む形になります。
```

### 開発比率・コードを書くか

```text
手を動かす比率は案件フェーズによりますが、実装・プロダクト化まで自分で進められることが前提です。PMだけ・営業だけではなく、技術で前に進める方を想定しています。
```

### カジュアル面談の内容

```text
選考ではなく相互理解の場です。FDEの役割、メダップのいまのフェーズ、（氏名）さんのご経験との接点を30〜45分ほどお話しします。ご質問は何でもどうぞ。
```

### 業務委託・副業可否

§3-F を使う。短答だけ必要なとき:

```text
FDEは正社員採用を想定しており、業務委託・副業枠での関わりは現時点ではお受けできません。将来的なご縁のためにカジュアルでお話しすることは歓迎です。
```

## 5. パーソナライズの入れどころ

| 相手の返信に含まれるもの | 差し替え例 |
|------------------------|------------|
| 具体的な関心（例：AI、現場伴走） | 「（関心のトピック）について、カジュアルで具体的にお話しできれば」 |
| 懸念（例：未経験、転職時期） | §4の該当Qを先に1文足す |
| 日程の希望 | §3-C。候補をそのまま引用 |
| 長文の質問 | 箇条書きで要点だけ答え、詳細は面談へ |
| 業務委託・副業希望 | §3-F。「情報交換」「中長期」で圧を下げる |
| 技術要件に不安・CS/PSの仕事に関心 | §3-G継続（FDE＝CS/PS＋エンジニアリング・キャッチアップ意欲重視）／§3-G区切り |

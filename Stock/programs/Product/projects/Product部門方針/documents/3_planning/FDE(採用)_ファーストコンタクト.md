# FDE（採用）転職サイト・ファーストコンタクト

> **用途**: 転職サイト（Wantedly、ビズリーチ、Green、YOUTRUST 等）で候補者へ送る **初回スカウト文のテンプレート**（ローカル専用）  
> **送信者**: 開発部門責任者の山田（FDEスカウトは山田名義）  
> **体裁の参考**: VPoT採用でCEO柳内が送ったファーストコンタクト（§2・送信者のみ異なる）  
> **JD**: [FDE(採用)_要件.md](./FDE(採用)_要件.md) / [Notion JD](https://www.notion.so/3731da5a7f3b812988a3d137b74dc772) / [採用ハブ](https://medup.notion.site/recruit)

| 文書 | 役割 |

**文字数**: 初回スカウトは **1,000字以内**（改行含む。§4テンプレは埋め込み後も上限内になるよう設計。送信前に媒体のカウンタで再確認）

## 1-1. 媒体別の装飾（コピペ時の注意）

スカウト本文は **プレーンテキスト前提** で書く（§4の `text` ブロックをそのまま貼る）。

| 使わない（多くの媒体で無効 or 崩れる） | 使ってよい |
|--------------------------------------|------------|
| `**太字**`、`*斜体*`、Markdown見出し | **URL**（そのまま1行） |
| 文字色・背景色 | 改行・空行 |
| HTMLタグ | 「」での強調、——での区切り |

- **Wantedly / ビズリーチ / Green** など: 装飾なしのことが多い → §4をそのまま
- リッチテキスト対応媒体でも、**初回スカウトは装飾なし**で統一しておけば媒体を問わず使える
- 強調したい語は **「初のFDE」** のように括弧・鍵括弧で足す（太字にしない）

### 段落の役割（FDE文にも流用）

| 順 | 役割 |
|----|------|
| ⓪ | タイトル（本文1行目・§3） |
| ① | 挨拶・送信者（開発部門責任者・山田） |
| ② | **個別化**（経歴・コメントから具体を1〜2点）→ なぜ連絡したか |

| ⑤ | **なぜ今そのポジションか**（組織のタイミング） |
| ⑥ | 募集URL |
| ⑦ | 締め（社会意義×その人）＋カジュアルCTA |
| ⑧ | 日程調整URL＋締めの一言 |

## 3. FDE用：タイトル（本文1行目）

**件名欄は使わない前提**。メッセージの **先頭1行** にタイトルを置き、2行目から挨拶（§4）。太字・色なし（§1-1）。

### 標準（これを使う）

```text
「ツールを導入する」だけでは届かない——メダップの FDE（Forward Deployed Engineer）募集について
```

別案は通常使わない（文字数が厳しいときのみ）: `【メダップ】FDE募集｜顧客課題の深掘り〜プロダクト化まで`

## 4. FDE用：送付文テンプレ（コピー用・**1,000字以内**）

**送信者**: 開発部門責任者の山田。括弧は候補者名・プロフィール具体のみ埋める（§5参照）。  
**装飾**: 太字・色なし（§1-1）。URLのみ行単位で貼る。  
**構成**: §3タイトル → 空行 → 挨拶〜（VPoT例と同じ流れ）  
**目安**: テンプレのまま約750字。具体①②を埋めても **1,000字以内**（2026-06-04時点）。送信前に媒体カウンタで確認。

```text
「ツールを導入する」だけでは届かない——メダップの FDE（Forward Deployed Engineer）募集について

はじめまして！メダップ、開発部門責任者の山田です。

（氏名）さんの（媒体名）の経歴（とコメント）を拝見し、（具体①）に非常に魅力を感じ、思わずご連絡しました。

メダップでは病院向けSaaSとCS/PSの伴走で、大病院の現場に入り込んで価値提供を続けてきました。そのなかで「ツールを導入する」だけでは課題解決まで至らない——と痛感しています。CS/PSとともに顧客課題を深掘りし、プロダクト化・実装・効果確認まで進める FDE を募集しています。
（氏名）さんのご経験は弊社にとって大きな力になると思います。ぜひ一度お話しできないでしょうか？

約14億円（プレシリーズBまで）を調達し、「医療版LayerX/Palantir」を目指すスタートアップです。病院特化AIとSaaSを組み合わせ、現場の課題解決まで伴走するフェーズに入っています。

開発部門ではCS/PSと一体でプロダクト化・実装・効果確認まで回し切ります。初のFDEとして、顧客デリバリー現場から「メダップのFDE」を一緒に形にしていただける方を迎えたいです。（具体②）のご経験をぜひご一緒いただきたいです。

募集概要：
https://www.notion.so/3731da5a7f3b812988a3d137b74dc772
https://medup.notion.site/recruit

「医療」領域で現場の一次情報からプロダクト・効果まで届けるには（氏名）さんの知見が不可欠です。ご興味があればオンラインでカジュアルにお話しさせてください。よろしくお願いします。

お返事お待ちしております。
```

**削ったもの（上限対策）**: US市場規模の1段落、採用ページの説明行の重複、締めの空行1つ、具体の長い括弧注釈（§5に移す）。

**まだ超えるとき**: ④の会社説明を1文にする → `調達済みスタートアップとして病院向けSaaSとAIで現場伴走を広げています。` に差し替え（約40字短縮）。

## 4-1. 「ツール導入だけでは届かない」前の言い換え案

伝えたい核はそのままに、**「SaaSコンパウンドの提供を拡大するなか」** だけ差し替える。

| 案 | 前文（→ そのあと「『ツールを導入する』だけでは…」） |
|----|--------------------------------------------------|
| **A（テンプレ採用）** | 病院向けSaaSとCS/PSによる伴走を重ね、多くの大病院の現場に入り込んで価値提供を続けてきました。そのなかで痛感しているのは、 |
| B | これまで多くの病院にSaaSと伴走支援を届けてきました。現場を見るほど、 |
| C | 導入病院を増やし、経営課題に向き合い続けてきたなかで、 |
| D | foro CRM 等のプロダクトとコンサルの伴走で、病院の変革支援を広げてきました。一方で、 |
| E（短め） | 病院の現場に深く入り込むほど、 |

**避けたい言い回し**: 「SaaSコンパウンド」「提供を拡大」——社内用語で、初見の候補者には届きにくい。

## 5. パーソナライズ差し替え（具体①・具体②）

§4の `（具体①）` `（具体②）` に貼る。合計で **1,000字以内** に収める（長いときは具体②を短く）。

| 候補者の傾向 | 具体①の例 | 具体②の例 |
|-------------|-----------|----------------|
| コンサル/SI × 開発 | ○○での顧客折衝とプロダクト開発リード | 顧客と同じテーブルで課題を深掘りし技術で前に進めてきた |
| スタートアップ | ○○での少人数での機能開発と顧客向き合い | 不確実ななか手を動かしながら価値を届けてきた |
| 効果まで見ている | リリース後の利用状況を見て改善まで回した | 「作って終わり」ではなく効果確認まで責任を持ってきた |
| AI活用 | AIを前提に開発スピードと品質を両立した | AI駆動開発の文脈で実装まで回せる |

## 6. 送る前に揃えるもの

```markdown
- [ ] 先頭行が§3のタイトルになっている（件名欄は空でよい）
- [ ] 氏名・媒体名
- [ ] 経歴・コメントから具体1〜2点（テンプレの（　）をすべて埋める）
- [ ] 山田の日程調整URL（なければカジュアルフォーム）
- [ ] 募集URL（Notion JD or 採用ハブ）
- [ ] **1,000字以内**（媒体カウンタ。超えたら §4「まだ超えるとき」）
- [ ] 本文に Markdown太字（`**`）や色指定を入れていない（§1-1）
```

**初回では書かない**: 確定年収、入社日、オファー。

## 7. 送付前チェック

- [ ] VPoT例と同程度に「なぜあなた」が具体で書けている
- [ ] FDEの定義（深掘り〜プロダクト化〜効果確認）が一文で伝わる
- [ ] 初のFDE・CS/PSとの共同が過不足なく触れられている
- [ ] 他社・現職へのネガティブ表現なし
- [ ] タイトルは1行目のみ（挨拶の前）。タイトル行を本文中に繰り返していない
- [ ] 太字・色・HTMLなし（`**` が残っていない）
- [ ] URLは単独行で貼り、クリックできる
- [ ] リンク・日程URLが有効
- [ ] **1,000字以内**（タイトル・URL・改行込み）
